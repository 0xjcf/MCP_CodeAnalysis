# Session Continuation Protocol

## Overview

This document outlines the protocol for continuing development sessions in the MCP Code Analysis project. The protocol ensures consistency, maintains context, and facilitates seamless continuation of development work.

## Prerequisites

- The project workspace must be in a clean state
- All previous changes must be committed
- The session continuation script must be executable

## Context Files

The session continuation process relies on several key context files:

1. **end-of-session.json**

   - Contains the final state of the previous session
   - Includes completed tasks, metrics, and decisions
   - Used as input for the next session's context
   - Generated by the session context generator

2. **ai-context.json**

   - Provides AI assistants with current codebase context
   - Includes file contents, search results, and project info
   - Updated during session continuation
   - Used by AI tools for informed assistance

3. **session_prompt.md**
   - Merged context from all sources
   - Used as the primary prompt for AI assistance
   - Updated at the start of each session
   - Contains current development status and goals

## Steps to Continue Session

1. **Initialize Session Continuation**

   ```bash
   chmod +x session-continuation.sh
   ./session-continuation.sh
   ```

2. **Review Generated Context**

   - Check `session_prompt.md` for the updated context
   - Verify all components are properly merged:
     - Session goals
     - Complexity analysis
     - Knowledge graph
     - Monetization status
     - Previous session context (from end-of-session.json)
     - AI context (from ai-context.json)

3. **Verify Analysis Results**

   - Review `complexity_analysis.json` for code complexity metrics
   - Check `knowledge_graph.json` for updated code relationships
   - Examine `monetization_status.json` for revenue opportunities
   - Validate `ai-context.json` for current codebase state
   - Confirm `end-of-session.json` contains previous session data

4. **Begin Development**
   - Use the updated context to continue development
   - Follow the established coding standards and guidelines
   - Maintain documentation as you work
   - AI assistants will use the context from ai-context.json

## Error Handling

If you encounter any issues during the session continuation:

1. **Script Execution Failures**

   - Check file permissions
   - Verify all required tools are installed
   - Review error messages in the console

2. **Context Merge Issues**

   - Verify all input files exist
   - Check file formats and schemas
   - Review the context merger logs
   - Ensure end-of-session.json is properly formatted
   - Validate ai-context.json structure

3. **Analysis Tool Failures**
   - Ensure all dependencies are installed
   - Check tool configurations
   - Review individual tool logs

## Best Practices

1. **Regular Updates**

   - Run the continuation script at the start of each session
   - Keep the context files up to date
   - Commit changes regularly
   - Verify AI context is current before seeking assistance

2. **Context Management**

   - Review and update session goals as needed
   - Monitor complexity metrics
   - Track monetization opportunities
   - Keep end-of-session.json current
   - Update ai-context.json when codebase changes

3. **Documentation**
   - Update relevant documentation
   - Maintain clear commit messages
   - Document any issues or blockers
   - Keep AI context documentation current

## Support

For issues or questions:

1. Check the project documentation
2. Review the error logs
3. Contact the project maintainers

## Notes

- The session continuation script creates backups of existing files
- All analysis results are stored in JSON format for easy parsing
- The context merger preserves previous session information
- AI context is automatically updated during session continuation
- End-of-session data is preserved between sessions
